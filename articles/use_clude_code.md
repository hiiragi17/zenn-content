# 始めに

Claude Codeに全てアプリを作成させるのを、
一回やってみたいと思っていました。
2026年は元旦にジョークアプリを出したい！
と思い、年末に作成していたのですが。
色々あって元旦を大いにすぎてしまいました。
もう1月が終わろうとしている時に、
今更初詣とか何？
という感じなのですが。
来年まで持ち越すほどでもないので、
ここで公開したいと思います……。

# 作成手順

Claude Codeだけで作成してみたいと言いつつ、
要件定義というか、
案出しはChat GPTで行いました。
デザインとかはGPTの方がセンスあるかなと思っているためです。
(あくまで個人の意見)

最初はターミナルでおみくじを引けたら面白いかな？
と考え、GPTと案を壁打ちしていました。
ただもう折角だしアプリ作成しちゃうかと思い、
そこでGPTに色々案を出してもらいました。
9個ぐらいの案の中に、
「AIがあなたの初詣先を勝手に決める」
というものがあり、
それが面白そうだと感じ、
そちらで案を詰めてもらいました。

最終的に詳細設計書をGPTに作ってもらい、
それをClaude Codeに渡すことで実装を進めてもらいました。
※2026だと長い間気が付かず、2025年で実装を続けていました

AIがあなたの初詣先を強制的に決めるとか言っていますが、
全部データを登録しています。
そんなことでAIを使えるか。
エンジニア向け運勢コメントも全部考えてもらいました。
神社は405種類、
運勢コメントは100種類あります。
組み合わせが多すぎる。

結局私がやったことは、
faviconの画像を置くことだけですね。
後は神社仏閣のリストを渡したぐらいです。
最初は全国の神社仏閣をランダムに表示させようとしたのですが、
折角なら全部京都の神社仏閣にしようと思い、
そちらを指定しました。
京都の神社仏閣は、
以前抹茶と神社。で神社仏閣リストを取得していたので、
そちらから流用しました。
実はまだ抹茶と神社。では全部の神社仏閣を実装できていないのですが、
このような形で日の目を見るとは……w

# アニメーションの限界

このアプリを遊んで頂いた方には、
わかると思うのですが、
ローディング演出がだいぶしょぼくなっています。
色々な案をClaude Codeには出してもらったのですが、
どれもカッコ悪いローディングの実装で……
ダサローディングを楽しむ方に振り切ることにしました。
これは私の中にアニメーションの引き出しがない故だとは思うのですが、
プロンプトだけの限界を感じました。
やっぱり普通のアプリのアニメーションは、
ちゃんとデザイナーさんがデザインしているんでしょうね。
おみくじ風演出はあまりにもありふれているので、
実装させなかったんですけど、
多分これもだいぶカオスなアニメーションになっていたと思います。
絵文字だけで表現しようとするからね。

# 短縮URLに関して

診断系アプリは、
結果をツイートして貰うことが一番の醍醐味だったりします。
今回は神社と運勢のどちらも共有して貰うための、
URLが必要だったのですが、
エンコードされてしまうため、
無茶苦茶に長くなってしまう問題が発生しました。
そのためBase64で短縮URLを実装してもらう機能を実装してもらっています。
43%の削減になっているようです。
他にも動的OGPの機能とか、
ツイッター共有機能とかもさっくり作ってくれました。
いや本当に凄い。
